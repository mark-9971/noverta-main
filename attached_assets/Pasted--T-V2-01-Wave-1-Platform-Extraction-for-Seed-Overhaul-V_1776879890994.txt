## Objective

Implement **T-V2-01 / Wave 1 Platform Extraction** from the Seed Overhaul V2 packet.

This wave should extract the stable seed platform substrate from the current monolith without intentionally changing the seeded district’s behavior.

The goal is to create the reusable structure that later waves will build on:
- deterministic config handling
- seed run metadata / batch identity
- extraction-safe utility modules
- scenario registry boundary
- event-loop boundary scaffolding
- post-run summary artifact scaffolding
- cleaner separation between:
  - platform
  - domain setup
  - scenarios
  - simulator
  - overlay

## Why now

The current seeder is already rich and valuable.
Wave 1 should not throw it away.
Wave 1 should make it decomposable.

This is the safest first implementation slice because it:
- reduces future complexity
- preserves current usefulness
- prepares the codebase for the real simulator and overlay work in later waves
