## Objective

Implement **T-V2-02 / Wave 2 Domain Foundation**.

This wave should extract and formalize the stable domain-building layer that sits above platform and below the later 9-month simulation engine.

The goal is to stop keeping core district/roster/IEP/reference setup as a giant monolith inside `seed-sample-data.ts`.

Wave 2 should create a clearer boundary for:

- district / school-year / school setup
- reference catalogs
- staff / guardian / student base creation
- scenario registry / assignment foundation
- IEP / goals / accommodations / service requirement base layer
- baseline schedule substrate inputs (not the full event-loop yet)

## Why now

W1 created the stable platform substrate.
W2 should now extract the **stable domain foundation** so later waves can build the simulator and demo overlay without continuing to rely on one giant file.

This is the correct next move because:
- it preserves the current seeder’s good domain richness
- it avoids premature simulator complexity
- it creates a cleaner seam for later event-loop and overlay work

## Scope

### In scope
At minimum, extract and formalize boundaries for the following:

1. **District / school structure**
   - district record shaping
   - school years
   - schools
   - school metadata

2. **Reference/catalog domain**
   - service catalog inputs
   - rate config inputs
   - programs / placements / supporting reference objects as appropriate

3. **Roster domain**
   - staff seeds
   - guardian/emergency contact seeds
   - student seeds
   - assignments / caseload-linking foundation

4. **Scenario foundation**
   - scenario archetype registry or definitions
   - assignment of scenarios to students
   - scenario metadata that later waves will consume
   - preserve the current valuable scenario richness; do not flatten it into generic randomness

5. **Clinical contract foundation**
   - IEP document base layer
   - goals
   - accommodations
   - service requirements
   - service-linked primitives needed later by simulation

6. **Baseline schedule substrate inputs**
   - enough extracted structure that later schedule/history simulation can be built cleanly
   - but do NOT do the full 9-month event-loop yet

### Out of scope
- no full 9-month simulator
- no full cadence emission for history tables
- no full demo overlay
- no full long-tail module pass
- no broad UI changes
- no cutover away from current seeder path
- no deletion of legacy seeders yet

## Required deliverables
