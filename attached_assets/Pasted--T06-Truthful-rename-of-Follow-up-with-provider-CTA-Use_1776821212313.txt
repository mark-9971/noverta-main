## Objective

Complete T06:
Rename the misleading Student Detail “Follow up with provider” CTA to truthful copy that reflects what the system actually does today.

Today, this action updates handling state / bookkeeping but does not send a real provider notification. The product should stop promising a real outreach action when the underlying workflow is not actually implemented.

## Why now

This is the smallest clean trust fix in the current roadmap.
It is explicitly the only narrow truth fix that fits cleanly alongside the just-finished T01–T05 chain.

In scope:
- find the Student Detail / Recommended Next Step CTA currently labeled “Follow up with provider”
- rename it to truthful language such as:

  - or another equally accurate label grounded in current behavior
- add short helper text / tooltip / secondary copy if needed so a user understands:
  - this updates internal handling state
  - it does not send a provider notification yet
- keep the change tightly bounded to this misleading CTA and its local explanatory text
