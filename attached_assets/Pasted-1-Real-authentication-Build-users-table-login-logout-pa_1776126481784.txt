1. Real authentication

Build:
	•	users table
	•	login/logout
	•	password hashing
	•	password reset
	•	invite flow for staff
	•	kill the demo role switcher

Definition of done:
	•	nobody can access the app without login
	•	staff accounts are real
	•	sessions expire properly

2. Backend RBAC

Build:
	•	auth middleware
	•	role checks on every API route
	•	school-scoped access
	•	caseload-based access where needed
	•	export restrictions

Definition of done:
	•	users only see what their role and school allow
	•	frontend role switching is gone
	•	API access is actually protected

3. Audit logging

Build:
	•	audit_logs table
	•	write logging for create/update/delete
	•	export logging
	•	capture user, timestamp, record, action, IP if available

Definition of done:
	•	every sensitive action is attributable
	•	you can inspect what changed and by whom

4. Real email sending

Build:
	•	email provider integration
	•	templates for:
	•	staff invites
	•	password reset
	•	parent notifications
	•	document delivery
	•	delivery/failure tracking

Definition of done:
	•	“sent” means actually sent
	•	failed sends are visible

5. Security hardening basics

Build:
	•	secure cookies
	•	HTTPS-only assumptions
	•	rate limiting on auth routes
	•	security headers
	•	CSRF protection if needed for your stack
	•	production-safe error handling
	•	separate dev vs prod config

Definition of done:
	•	obvious low-level security holes are closed

6. Basic backup / recovery plan

Build:
	•	automated database backups
	•	at least one tested restore flow
	•	written backup cadence
	•	environment separation

Definition of done:
	•	you know how to recover from a mistake or outage
