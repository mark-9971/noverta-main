C. Highest-Value Completed Improvements
1. Support Intensity Score — Gives the clinical team an auditable, transparent snapshot of how intensive a student's support package is. Nothing like this exists in competing tools. Clinically differentiating.

2. ABA Tab UX Overhaul (this session) — The FBA context bar specifically. Previously, a BCBA in ABC Data mode had no idea which FBA they were looking at. Now it's always visible. The auto-advance to ABC tab on FBA select cuts 2 clicks from the most common workflow.

3. Reinforcer Inventory — Gives each student a structured, living document of what actually works for them. No SPED compliance platform has this. Direct demo value for a BCBA audience.

4. ABA Navigation Restructure — The 6-domain clinical structure (Learners / Sessions / Programs / Analysis / Reporting / Supervision) is the right framework. It mirrors how BCBAs think about their day.

D. Current Biggest Remaining Gaps
1. No "Is this BIP working?" view — There is no visualization showing behavior frequency before and after BIP implementation date. A BCBA cannot look at this platform and see whether an intervention is effective. This is the most glaring clinical gap.

2. No BIP review cycle tracking — BCBAs are required to review BIPs annually (minimum). There is no review_due_date field, no alert, no workflow. This is a compliance risk for pilot districts.

3. No prompt fading timeline — PromptDependencePanel shows current prompt level. It cannot answer "has this student's prompt level changed in the last 8 weeks?"

4. FBA→BIP auto-bridge is informational, not workflow — ABC data is visible alongside the BIP editor but doesn't pre-populate anything. The bridge is display-only.

5. Behavior Data is in two disconnected places — BehaviorsTab (in Programs & Data) has trend graphs. AbcDataPanel (in Assessments) has observation records. Neither talks to the other. A BCBA asking "is this behavior reducing since we implemented the BIP?" has to mentally correlate data from two separate tabs.

E. Recommended Next 5 Tasks (Highest Leverage First)
1. BIP effectiveness overlay — Add a "BIP implemented on [date]" vertical marker to the behavior trend chart in BehaviorsTab. One line of reference. Clinically transforms the graph from "history" to "evidence."

2. BIP review cycle tracking — Add reviewDueDate and lastReviewedAt to the BIP schema. Show a badge ("Review due in 12 days") in the Assessments tab student header. Wire an alert if overdue.

3. Prompt fading timeline chart — A small line chart per program target showing prompt level (encoded as 0–5) over the last N sessions. Answers "are we fading?" in 3 seconds. Data is already collected; just needs visualization.

4. "Create BIP from FBA" button — In FbaListPanel, add a button that creates a new BIP pre-populated with: targetBehavior from the FBA, hypothesizedFunction from obsSummary, antecedentStrategies seeded from top ABC antecedents. Saves 20 minutes per BIP.

5. ABC data visualization — Replace the raw ABC observation table in AbcDataPanel with a bar chart (top 5 antecedents, top 5 consequences). Clinically useful for function hypothesization and faster to read in an IEP meeting than a data table.
