Goal:
Make sure school-calendar exceptions also flow correctly into compensatory / makeup / exposure logic, so closure days and early-release days do NOT create false owed minutes, false missed-service exposure, or inflated financial/cost views.

This is a read/business-logic alignment slice. Do not broaden into a workflow redesign.

B. Align business logic with school-calendar exceptions

1. closure day for a school/date:
- should not create expected service obligation for that date
- should not generate compensatory time purely because no service occurred that day
- should not inflate missed-minute exposure/cost/liability for that date

2. early_release day:
- should only create obligation/exposure based on the currently implemented fallback rule already used in the shared minute-progress logic
- do not introduce a second competing rule
- if the current shared fallback is 0.5 day weight, downstream consequence logic must use that same rule

3. requiredMinutes / IEP contract remains unchanged overall
- the alignment should affect expected-by-now / shortfall / owed logic, not rewrite the underlying IEP requirement itself

- Reuse the shared school-calendar logic from prior slices
- Do not duplicate closure/early-release math inside compensatory or finance routes

- the values should now reflect school-calendar-adjusted expectation logic

- avoid clutter
