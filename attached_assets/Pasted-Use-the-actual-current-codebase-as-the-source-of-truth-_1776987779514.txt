## Compatibility requirements

For runtime/client persisted identifiers, do not break existing users.

If renaming:

* localStorage keys
* BroadcastChannel names
* window event names
* debug globals

then add a temporary compatibility shim where appropriate.

Examples:

* read old `trellis.*` localStorage keys once and migrate to `noverta.*`
* preserve old event listener temporarily while emitting the new event
* support old BroadcastChannel name only if needed during transition
* document which shims are temporary and when they can be removed

Do not silently orphan existing local state.
