# What I want you to optimize for

## 1. Fewer clicks, more clarity

The ABA package should feel streamlined and intentional, not scattered across too many disconnected screens.

## 2. Role-based efficiency

Different users need different “front doors”:

* BCBA / clinical lead
* para / direct staff
* teacher / school staff
* coordinator / admin

The nav and landing experience should reflect that where appropriate.

## 3. Workflow-first design

Design around the actual ABA workflow:

* understand the learner
* review active programs / BIP / supports
* run session / collect data
* review performance
* update programming
* analyze trends
* generate report
* supervise fidelity / IOA / staff implementation

## 4. Clinically serious organization

The IA should reflect real ABA work:

* programs
* targets
* sessions
* behavior support
* graphs
* reports
* runbook
* analytics
* supervision

## 5. Preserve power while reducing clutter

Do not dumb it down.
The goal is **easy to use but information-rich**.

---

# Design principles

## A. Organize by jobs-to-be-done, not by database table

Avoid nav labels that mirror schema or dev structure.
Use labels and groupings that make sense to clinicians and school staff.

## B. Minimize context switching

Related tasks should live close together.
Live data collection, recent performance, program instructions, and learner supports should not require bouncing across many pages.

## C. Make “what do I do with this student?” obvious

A staff member should be able to quickly understand:

* what programs are active
* how to run them
* what prompts/supports are needed
* what problem behaviors/supports matter
* what reinforcers/preferences matter
* what changed recently

## D. Analysis should be easy to reach from action

Graphs and analytics should not feel buried.
Users should be able to move naturally from:
program → session data → graph → decision

## E. Reporting should emerge from real data

Progress reports and summaries should feel downstream from programming and data, not like isolated documents.

Make the IA/sidebar/workflow changes.

Make sure the redesigned ABA area strongly supports:

* start session

## Phase 5 — Improve the user flow around learner understanding

# Important implementation guidance

## Do not just rename things

This must be a real workflow redesign, not superficial relabeling.

## Do not overfragment

Avoid too many tiny menu items.

## Do not bury live data collection

Live data and session workflows should be extremely easy to reach.

## Do not bury the runbook

If the runbook / learner summary is strong, it should be prominent.

## Do not bury graphs

Graphs and recent performance must be easy to reach from the learner/program workflow.

## Do not bury BIP

Behavior support should feel integrated with learner understanding and data, not isolated.
