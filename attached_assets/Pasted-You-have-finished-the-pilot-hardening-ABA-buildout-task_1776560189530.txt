## Your job
For every task we assigned in this buildout, determine whether it is:

- Completed and verified

# Required process

For each task, do this:

## 1. Restate the original task goal
Briefly say what the task was supposed to accomplish.

## 2. Audit what was actually built
Show:
- files changed
- schema changes
- routes/APIs changed
- UI/components changed
- logic changes

## 3. Determine whether the implementation is correct
Check:
- is the correct data source being used?
- is the workflow actually complete?
- is the UI wired to real logic?
- are there edge cases or false states?
- is it clinically / operationally correct where relevant?

## 4. Verify
Run the most relevant verification available:
- build
- typecheck
- tests
- route checks
- manual workflow checks
- seeded/demo-data checks
- role-based checks
- mobile checks if relevant

Do not skip verification unless truly impossible.
If impossible, say exactly why.

## 5. Final judgment for that task
Mark one:
- completed and verified
- completed but weakly verified
- partial
- incorrect / needs rework
- blocked

Then explain why in 2–5 sentences.

Answer these directly:

- Which need rework before pilot?

For every assigned task, include:

- target goal
- status
- verification performed

Call out anything that looks done but is not actually correct.
Examples:
- wrong data source
- empty charts due to missing seed data
- UI-only implementation without real workflow

## 4. Regression findings

## 5. Demo readiness after this work
Specifically say:
- what is safe to demo
- what should still be hidden
- what would break trust if clicked

## 6. Pilot readiness after this work
Specifically say:
- what is truly pilot-ready
- what is close
- what is not ready
- what 5 items most still block a narrow pilot

## 7. Final rework list
Give me the exact tasks that must be redone or tightened next.

# Important rules
- Be skeptical.
- Verify instead of assuming.
- “Touched code” does not equal complete.
- “Visible UI” does not equal working workflow.
- “No errors” does not equal clinically or operationally correct.
- If a feature depends on seed/demo data to look good, say so.
- If a feature works only partially, say so.

This is a final truth pass.
I want accuracy, not encouragement.
